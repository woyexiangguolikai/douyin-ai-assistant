## 一、概述

### 1.1 产品概述及目标

抖音娱乐主播在直播时面临三大痛点：弹幕太多看不过来、不知道回什么导致冷场、没有故事素材可讲。

**产品定位**：一款运行于 Windows 桌面的 Electron 工具，实时抓取抖音直播间弹幕，由 DeepSeek AI 根据主播人设自动生成回复话术和故事素材，主播可边看边用。

**量化目标**：

| 指标 | 当前值 | 目标值 | 达成时间 |
|------|--------|--------|----------|
| 主播平均互动频次 | 每30秒1次 | 每15秒1次 | 使用后第1周 |
| 直播冷场时间占比 | ~20% | <5% | 使用后第1周 |
| 故事类弹幕回复质量满意度 | — | ≥80% | 使用后第1月 |

### 1.1.5 范围说明

**本期做**：弹幕实时采集、AI实时生成回复话术和故事、主播人设配置、管理后台基础框架。

**本期不做**：移动端App、多平台同时采集（仅抖音）、AI语音合成朗读、数据分析看板、付费订阅。

### 1.2 名词说明

| 术语 | 说明 |
|------|------|
| 弹幕（Danmaku） | 抖音直播间观众发送的实时滚动评论 |
| 人设（Persona） | 主播在直播中扮演的角色设定，含性格、语气、禁忌话题等 |
| AI提词 | AI生成的、供主播朗读或参考的话术文本 |
| 玻璃态（Glassmorphism） | 界面设计风格：半透明背景 + backdrop-filter模糊 + 霓虹边框 |
| 采集方案（Capture Method） | 弹幕抓取的技术方案，当前支持Playwright浏览器和WebSocket两种 |

### 1.3 角色及权限

| 角色 | 权限 | 说明 |
|------|------|------|
| 主播（Host） | 使用桌面客户端所有功能 | 直接操作者 |
| 管理员（Admin） | 管理后台全部权限：创建/管理直播间、管理人设、管理运营人员 | 团队管理者 |
| 运营（Operator） | 管理后台被分配的直播间配置权限 | 辅助运营 |

### 1.4 文档阅读对象

- 产品经理：确认范围和交互逻辑
- 前端开发：React组件实现
- 后端开发：Electron主进程 + API接口
- 测试：功能验收

## 二、产品描述

### 2.1 产品需求描述

桌面客户端通过浏览器自动化连接抖音直播间，实时采集弹幕，经智能过滤后送入DeepSeek AI，AI根据预设主播人设生成回复话术，在界面右侧区域以淡入动画展示。主播可一键复制话术朗读，或根据故事类回复即兴发挥。

管理后台（Web端）供团队管理员统一配置多个直播间的人设、AI参数和运营人员。

### 2.1.5 页面流转全景图

`
桌面客户端（Electron）

## 三、功能需求

### F01 · Playwright弹幕采集

**功能优先级**：P0
**页面流转**：桌面客户端 → 输入房间号 → 点击连接 → 主界面弹幕列表

---

#### 👤 客户可读区

**用户操作步骤**：
1. 打开桌面客户端，在顶部工具栏输入抖音直播间房间号
2. 点击「连接直播间」按钮，系统自动启动Edge浏览器
3. 浏览器自动导航至抖音直播间页面
4. 主界面左侧弹幕列表开始实时显示弹幕内容（每条包含用户名+内容）
5. 点击「断开」关闭连接

**输出/后置条件**：弹幕实时流持续采集，直至主播主动断开或网络异常

**用户故事**：作为主播，我希望打开工具就能自动采集直播间弹幕，以便我不需要手动刷新页面

---

#### 🔧 开发可读区

**输入/前置条件**：本地已安装Edge浏览器

**需求描述**：
1. 用户在工具栏输入房间号，点击连接
2. 主进程调用Playwright启动Edge（headless: false但窗口隐藏），导航至抖音直播间
3. 注入JS脚本监听WebSocket消息（wss://broadcast/room/*），解析protobuf弹幕数据
4. 每条弹幕提取：用户名、内容、类型（chat/gift/enter/system）
5. 通过IPC发送至渲染进程（channel: danmaku-received）
6. 渲染进程将弹幕追加到列表顶部

**异常场景**：

| 场景 | 处理方式 | 提示文案 |
|------|---------|---------|
| 房间号不存在 | 返回错误，不启动浏览器 | "房间号不存在或直播间未开播" |
| Edge未安装 | 自动跳过Playwright，尝试WebSocket | — |
| 采集超过60s无数据 | 状态指示灯变黄 | — |
| 浏览器崩溃 | 自动尝试重启3次 | "采集异常，正在自动恢复..." |

### F02 · AI实时回复生成

**功能优先级**：P0
**页面流转**：弹幕到达 → 自动送入AI → 右侧面板淡入展示

---

#### 👤 客户可读区

**用户操作步骤**：
1. 在设置面板填入DeepSeek API Key并保存
2. 开启AI开关（默认开启）
3. 弹幕到达后，右侧面板显示「AI正在思考」动画（三个跳动的绿色圆点）
4. AI生成完毕后，完整回复以自下而上的淡入动画出现
5. 回复卡片展示：触发弹幕（虚线连接）、生成文本、生成时间、复制按钮
6. 点击「复制」将文本复制到剪贴板

**输出/后置条件**：回复保存在历史列表中，可导出为TXT

**用户故事**：作为主播，我希望AI能在弹幕到达后自动生成自然的话术回复，以便我在直播中快速响应观众

---

#### 🔧 开发可读区

**输入/前置条件**：DeepSeek API Key已配置、AI开关已开启

**需求描述**：
1. 弹幕到达后进入防抖队列（1500ms内多条弹幕合并为一批）
2. 主进程将弹幕批送AI服务，构建提示词（含主播人设、弹幕上下文、回复模式）
3. 调用DeepSeek API（非流式），等待完整回复
4. 回复生成期间，IPC发送eply-start事件至渲染进程，触发「AI正在思考」动画
5. 回复完成后，IPC发送eply-stream事件（done: true），触发前端淡入动画展示
6. 渲染进程使用CSS @keyframes fade-in（opacity 0→1 + translateY 8px→0，0.4s ease-out）

**数据字典**（AI回复历史记录）：

| 字段名 | 类型 | 必填 | 校验规则 | 默认值 | 说明 | 示例值 |
|--------|------|------|---------|--------|------|--------|
| id | varchar(32) | 是 | 唯一 | 自动生成 | 回复记录ID | h_1699923456789 |
| text | text | 是 | 非空 | — | AI生成的完整回复文本 | "家人们，这个问题问得好..." |
| time | varchar(8) | 是 | HH:MM:SS | — | 生成时间 | 14:30:25 |
| danmaku | json | 否 | — | [] | 触发该回复的弹幕列表 | [{"username":"..."}] |

**异常场景**：

| 场景 | 处理方式 | 提示文案 |
|------|---------|---------|
| DeepSeek API Key无效 | 状态指示变红，不发送请求 | "API Key 无效，请在设置中重新配置" |
| API返回错误（限流/余额不足） | 记录错误，跳过本次请求 | —（不打扰主播） |
| API超时（30s） | 放弃本次请求，不影响后续 | —（不打扰主播） |
| 弹幕队列溢出 | 丢弃最早未处理的批次 | —（不打扰主播） |
| 回复内容为空 | 不显示回复卡片 | — |

## 四、非功能需求

### 4.1 安全与合规

- DeepSeek API Key本地存储，不传输至第三方服务器
- JWT token有效期7天，过期自动清除
- 密码使用bcrypt加盐哈希（salt rounds=10）
- 禁忌话题可在人设中配置（默认屏蔽政治、宗教、涉黄、人身攻击）

### 4.3 性能需求

| 指标 | 要求 |
|------|------|
| 弹幕采集延迟 | <500ms |
| AI回复生成时间 | <10s（DeepSeek API正常响应） |
| 应用启动时间 | <3s |
| 内存占用 | <300MB（含Electron + Playwright） |

### 4.4 数据库设计

使用sql.js（SQLite编译为WebAssembly），本地存储，无需额外数据库服务。

**本地数据库表**（桌面客户端）：
- settings：应用配置（房间号、API Key、采集方式等）
- personas：人设模板
- filter_rules：过滤规则
- session_history：弹幕/AI回复历史

**后端数据库表**（管理后台）：
- users：用户账号
- rooms：直播间配置
- personas：人设配置
- ai_settings：AI参数配置
